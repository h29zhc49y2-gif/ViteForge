# ViteForge

## 任务目标

根据项目实际文件结构和用户需求，生成可直接使用的完整 `vite.config.ts`。

## 执行流程

### Step 1：检测项目环境

运行检测脚本：
```bash
python scripts/project_detect.py "<项目根目录>"
```

获取：框架类型/是否TS/包管理器/现有 Vite 插件/package.json 内容

### Step 2：理解用户需求

从用户输入提取：
- 技术栈（React/Vue/Svelte，默认从检测结果推断）
- 别名配置（如 @/ → src/，默认自动推断）
- 代理目标（如 /api → http://localhost:8080）
- 特殊场景（库模式/MPA多页/SSR/Electron）
- 额外插件（visualizer/compression/等）

### Step 3：生成配置

调用生成脚本：
```bash
python scripts/generate_vite_config.py --root <项目根目录> [选项...]
```

### Step 4：输出与检查

- 输出生成的 `vite.config.ts` 内容（带注释）
- 列出需要安装的依赖（npm/pnpm/yarn 命令）
- 补充使用说明（npm run dev / npm run build）

## 核心规则

### 规则 1：别名自动推断

根据项目结构推断别名，常见模式：

| 目录存在 | 别名配置 |
|---------|---------|
| src/ | @/ → ./src |
| src/ + components/ | @/ → ./src |
| lib/ | @lib/ → ./lib |
| packages/ | monorepo 多入口 |

若 package.json 已有 paths 配置，优先使用。

### 规则 2：代理配置

简单代理：
```ts
server: {
  proxy: {
    '/api': 'http://localhost:8080'
  }
}
```

复杂代理（多个后端/WebSocket）：
```ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    },
    '/ws': {
      target: 'ws://localhost:8080',
      ws: true
    }
  }
}
```

### 规则 3：插件按需装配

| 场景 | 推荐插件 | 需要安装 |
|------|---------|---------|
| React | @vitejs/plugin-react | npm i -D @vitejs/plugin-react |
| Vue 3 | @vitejs/plugin-vue | npm i -D @vitejs/plugin-vue |
| Svelte | @sveltejs/vite-plugin-svelte | npm i -D @sveltejs/vite-plugin-svelte |
| TypeScript | (内置) | 无需额外插件 |
| 路径别名 | (内置 resolve.alias) | 无需额外插件 |
| 打包分析 | rollup-plugin-visualizer | npm i -D rollup-plugin-visualizer |
| 压缩 | vite-plugin-compression | npm i -D vite-plugin-compression |
| SVG 组件 | vite-plugin-svgr | npm i -D vite-plugin-svgr |
| 自动导入 | unplugin-auto-import | npm i -D unplugin-auto-import |
| 组件自动注册 | unplugin-vue-components | npm i -D unplugin-vue-components |
| PWA | vite-plugin-pwa | npm i -D vite-plugin-pwa |
| WindiCSS | vite-plugin-windicss | npm i -D vite-plugin-windicss |
| Unocss | unocss | npm i -D unocss |
| ESLint | vite-plugin-eslint | npm i -D vite-plugin-eslint |
| Markdown | vite-plugin-md | npm i -D vite-plugin-md |
| 环境变量 | (内置 loadEnv) | 无需额外插件 |

### 规则 4：生成配置必须包含的注释

每行关键配置都要注释作用，帮助开发者理解：
```ts
// ===== 路径别名 =====
resolve: {
  alias: {
    '@': path.resolve(__dirname, './src'),  // @/ 指向 src 目录
  }
}
```

### 规则 5：完整配置结构

```ts
import { defineConfig, loadEnv } from 'vite'
import react from '@vitejs/plugin-react'
import { resolve } from 'path'

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd())  // 加载 .env 文件
  
  return {
    // ===== 插件 =====
    plugins: [react()],
    
    // ===== 路径别名 =====
    resolve: {
      alias: { '@': resolve(__dirname, './src') }
    },
    
    // ===== 开发服务器 =====
    server: {
      port: 3000,
      proxy: { '/api': 'http://localhost:8080' }
    },
    
    // ===== 构建配置 =====
    build: {
      outDir: 'dist',
      sourcemap: mode === 'development',
    }
  }
})
```

## 特殊场景

### 库模式
```ts
build: {
  lib: {
    entry: resolve(__dirname, 'src/index.ts'),
    name: 'MyLib',
    formats: ['es', 'umd'],
    fileName: (format) => `my-lib.${format}.js`
  },
  rollupOptions: {
    external: ['react', 'react-dom'],  // 不打包进库
    output: { globals: { react: 'React' } }
  }
}
```

### MPA 多页面
```ts
build: {
  rollupOptions: {
    input: {
      main: resolve(__dirname, 'index.html'),
      admin: resolve(__dirname, 'admin/index.html'),
    }
  }
}
```

### Electron
```ts
// 需要 vite-plugin-electron
import electron from 'vite-plugin-electron'
plugins: [
  react(),
  electron([{
    entry: 'electron/main.ts',
  }, {
    entry: 'electron/preload.ts',
  }])
]
```

## 环境变量约定

自动识别 `.env` 文件：
- `.env` — 所有环境
- `.env.development` — dev
- `.env.production` — production

```ts
// 使用 define 暴露给客户端
define: {
  __APP_VERSION__: JSON.stringify(env.VITE_APP_VERSION),
}
```

## 构建优化默认值

- `build.target: 'es2020'` — 兼容现代浏览器
- `build.chunkSizeWarningLimit: 1000` — 提高分块警告阈值
- `build.rollupOptions.output.manualChunks` — 自动拆分 node_modules
```
